📝 Project Overview

This framework is built using Playwright with TypeScript and follows the Page Object Model pattern to automate UI workflows efficiently. The project emphasizes reliability, reusability, and CI/CD readiness.

✅ Key Features

  • Playwright automation using TypeScript
  • Robust Page Object Model structure
  • Retry utility for resilient test steps
  • Dynamic waits to avoid flakiness
  • Logging and console tracing for debugging
  • CI/CD pipeline using Jenkins

✅ Configurations

  • baseURL: 'http://jupiter.cloud.planittesting.com'
  • headless: false
  • timeout:50000
  • Browser: Chromium (But can be enabled other browsers or can pass through commandline)

✅ Tools

  • Jenkins, Node.js, npm, Git

🧪 Test Cases

Test Case 1: Contact Form Validation

  1. From the home page, navigate to the Contact page
  2. Click the Submit button
  3. Verify error messages for required fields
  4. Populate mandatory fields: Forename, Email, Message
  5. Verify that the error messages disappear

Test Case 2: Successful Contact Submission

  1. From the home page, navigate to the Contact page
  2. Fill in the mandatory fields
  3. Click Submit
  4. Verify the success message appears
  5. This test runs 5 times to validate stability

Test Case 3: Shopping Cart Price Validation

  1. Buy 2 Stuffed Frogs, 5 Fluffy Bunnies, 3 Valentine Bears
  2. Navigate to the cart page
  3. Validate individual product prices
  4. Validate subtotal for each product line
  5. Ensure the total equals the sum of all subtotals

🏛️ Framework Architecture

Framework Layers

  • Test Layer - Contains Playwright tests using the Page objects
  • Page Layer - Encapsulates UI elements and actions per page
  • Utils Layer - Common utilities like retry and wait logic

Playwright Architecture:

  • Browser Context & Page Objects managed per test run
  • Auto-waiting mechanisms built-in
  • Parallel execution and trace recording supported

🛠️ Best Practices Followed

  • POM architecture for separation of concerns
  • Retry logic in utility for flaky elements
  • Use of Playwright's dynamic waiting instead of fixed timeouts
  • Reusability and modularization of test scripts
  • Console logging and step tracing for clarity

🔁 CI/CD Integration with Jenkins

Jenkins Setup

  1. Install Jenkins and required plugins
  2. Ensure Jenkins agent has Node.js, npm, Git, and Playwright
  3. Configure using the provided Jenkinsfile

Jenkinsfile

pipeline {
  agent any
  tools {
    nodejs 'NodeJS'
  }
  stages {
    stage('Install') {
      steps {
        sh 'npm install'
      }
    }
    stage('Test') {
      steps {
        sh 'npx playwright install'
        sh 'npx playwright test'
      }
    }
  }
  post {
    always {
      archiveArtifacts artifacts: 'test-results/**/*.*', allowEmptyArchive: true
    }
  }
}
